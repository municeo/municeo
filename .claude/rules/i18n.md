# Internationalization (i18n)

All user-facing text goes through Symfony Translator. Routes are localized. The app is multi-tenant and locale-aware.

## Never hardcode user-facing strings

```twig
{# DO — translation key #}
<h1>{{ 'report.create.title'|trans }}</h1>
<p>{{ 'report.error.daily_limit'|trans }}</p>
<button>{{ 'action.submit'|trans }}</button>

{# DON'T — hardcoded French #}
<h1>Créer un signalement</h1>
<p>Limite journalière atteinte</p>
```

## Translation keys in English, descriptive, dot-separated

```yaml
# DO — translations/messages.en.yaml
report:
    create:
        title: "Create a report"
        success: "Report submitted successfully"
    error:
        daily_limit: "Daily limit reached"
        cooldown: "Please wait before submitting again"
        duplicate: "A report already exists in this area"
action:
    submit: "Submit"
    cancel: "Cancel"

# DON'T — French keys, flat structure
creer_signalement: "Create a report"
limite: "Daily limit reached"
```

## Localized routes with {_locale} prefix

```yaml
# DO — config/routes.yaml
controllers:
    resource: ../src/Infrastructure/Http/Controller/
    type: attribute
    prefix: /{_locale}
    requirements:
        _locale: en|fr

# Route definition in controller
#[Route('/report/new', name: 'report_create')]

# Generates: /en/report/new, /fr/report/new
```

```php
// DO — route names are English, locale-agnostic
#[Route('/report/{id}', name: 'report_show')]
#[Route('/agent/report/{id}/validate', name: 'agent_report_validate')]

// DON'T — French route paths
#[Route('/signalement/{id}', name: 'signalement_voir')]
#[Route('/agent/signalement/{id}/valider', name: 'agent_signalement_valider')]
```

## Localized route paths via translation

```yaml
# DO — translations/routes.fr.yaml
report_create: /signalement/nouveau
report_show: /signalement/{id}
agent_report_validate: /agent/signalement/{id}/valider

# translations/routes.en.yaml
report_create: /report/new
report_show: /report/{id}
agent_report_validate: /agent/report/{id}/validate
```

## Error messages from controllers through translator

```php
// DO — translate in controller/template, not in domain
catch (ReportDailyLimitExceededException) {
    $this->addFlash('error', $this->translator->trans('report.error.daily_limit'));
    throw new TooManyRequestsHttpException();
}

// DON'T — hardcoded French error message
catch (ReportDailyLimitExceededException) {
    $this->addFlash('error', 'Limite journalière atteinte');
}
```

## Domain exceptions stay in English, translation at the edge

```php
// DO — domain throws English, controller translates
// In Domain:
throw new ReportDailyLimitExceededException('Daily report limit exceeded');

// In Controller:
catch (ReportDailyLimitExceededException) {
    $message = $this->translator->trans('report.error.daily_limit');
}

// DON'T — domain exception in French
throw new ReportDailyLimitExceededException('Limite journalière atteinte');
```

## Email templates localized

```php
// DO — use translator for email subject/body
$email = (new TemplatedEmail())
    ->subject($this->translator->trans('email.report_resolved.subject', locale: $userLocale))
    ->htmlTemplate('emails/report_resolved.html.twig')
    ->context(['locale' => $userLocale]);

// DON'T — hardcoded French subject
$email = (new TemplatedEmail())
    ->subject('Votre signalement a été résolu');
```
